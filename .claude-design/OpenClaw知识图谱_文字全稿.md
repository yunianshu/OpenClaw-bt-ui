### 第34页：缺点 1：Token 成本“吞金兽”
- 日均费用：几十到几百到上千元
- OpenClaw 自身免费，但需调用大模型 API
- 多次调用模型进行规划、执行、验证
- Claude 等 API 费用高昂

---

### 第35页：缺点 2：技术门槛高
- 部署过程极其复杂 — 通常需要命令行操作
- 连专业人士都可能耗费数小时调试
- 令人望而却步的技术门槛

---

### 第36页：缺点 3：安全风险
- ⚠ **高系统权限**：一旦出错可能导致数据丢失
- ⚠ **恶意利用**：被恶意利用的风险
- ⚠ **意图理解**：安全审计中关键项目得分曾为 0%

---

### 第37页：缺点 4：稳定性不足
- 快速迭代 — 作为快速迭代的开源项目稳定性不足
- 版本升级失效 — 某次升级曾引发大规模功能失效
- AI 幻觉 — 可能编造事实 / 产生幻觉

---

### 第38页：缺点 5：“开箱即用”误区
**并非立即可用**：
部署后需安装配置各种技能 / 执行效果依赖用户编写 SOP

**隐性成本高**：
部署学习成本高 / 普通用户可能放弃 / 关键结果需人工复核
